New features and improvements shipped every week.

## April 2026

<Update label="v0.92 - Apr 26, 2026" tags={["Improvements", "Settings"]}>
  ## Workspace Refresh & Node-Level Credits

  A cleaner workspace with consistent backgrounds and tabbed settings, plus credits now meter and resume per node so long workflows do not stall silently when credits run low.

  ### Improvements

  - Workspace surfaces refreshed with consistent backgrounds, pill-style tabs, and a tabbed settings panel
  - Runtime picker added to the home build prompt and single-panel build view
  - Integrations catalog now focuses on connecting one toolkit at a time, with clearer setup steps
  - Node-level credit metering lets workflows pause and resume cleanly when credits run low
</Update>

<Update label="v0.91 - Apr 23, 2026" tags={["Improvements", "Reliability"]}>
  ## Self-Healing Runs

  When a run fails, Decisional can diagnose the cause, retry transient issues, patch code-level bugs, or ask the workflow owner for missing details before resuming the run.

  ### Improvements

  - Self-healing for failed runs with automatic triage, patching, retries, and human escalation
  - Slack clarification requests when a missing rule, mapping, or threshold blocks a run
  - Reminder pings at 60 minutes, 2 hours, and 24 hours for stalled repair sessions
  - Runtime picker redesigned as a logo dropdown across build modes
  - Managed filter layer for triggers so incoming events can be narrowed before they launch work
</Update>

<Update label="v0.90 - Apr 22, 2026" tags={["New features", "Agents"]}>
  ## Multiple Modes, One Agent

  Agents can switch between Chat, Operator, and Build modes from one picker without losing the current thread.

  ### Improvements

  - Mode switcher per agent for moving between Chat, Operator, and Build
  - Unified promptbox across modes
  - Smart thread resumption when switching modes mid-conversation

  Learn more in [Dex and Agent Modes](/agents/modes).
</Update>

<Update label="v0.89 - Apr 21, 2026" tags={["New features", "Public runs"]}>

  Public run pages gained a pinned chat overlay, authenticated viewers gained an in-app assistant overlay, and workflow viewing became tighter around run context.

  ### Improvements

  - Public run chat overlay pinned to the right of the page
  - AI Assistant chat overlay for authenticated viewers
  - Workflow viewer keeps run context in the viewer and folds node tabs into one Configurations panel
  - Sidebar pinned section with toolkit tiles and recent-agent ordering
</Update>

<Update label="v0.88 - Apr 20, 2026" tags={["Improvements", "Approvals"]}>

  ### Improvements

  - Growth plan now includes 75,000 credits per month
  - Usage and Tool Calls merged into Dashboard tabs with per-user attribution and account type columns
  - Approvals hardened to prevent double-approval, add a bypass toggle for solo runs, and surface pending state more clearly

  Learn more in [Approvals and Policy](/agents/approvals).
</Update>

<Update label="v0.85 - Apr 10, 2026" tags={["New features", "Approvals"]}>
  ## Human Approvals Anywhere In Your Workflow

  Workflows can pause for human review with Gate nodes, including public approval links for stakeholders who do not need a Decisional account.

  ### Improvements

  - Add a Gate node anywhere in a workflow to pause execution until a human approves or rejects
  - Set quorum rules so any one approver or every listed approver must sign off
  - Share public approval links with the full decision context

  Learn more in [Approvals and Policy](/agents/approvals).
</Update>
