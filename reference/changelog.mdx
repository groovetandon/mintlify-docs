<Update label="April 26, 2026" tags={["Improvements", "Settings"]}>
  ## Workspace Refresh and Node-Level Credits

  A cleaner workspace experience with tabbed settings, more consistent surfaces, and more resilient metering for long-running workflows.

  - Refreshed workspace backgrounds, tabs, and settings layout
  - Runtime selection available from the build prompt and build view
  - Integration setup flow focused around one toolkit at a time
  - Node-level credit metering so long workflows can pause and resume cleanly when credits run low

</Update>

<Update label="April 23, 2026" tags={["Improvements", "Reliability"]}>
  ## Self-Healing Runs

  Failed runs now get a repair path instead of simply stopping. Decisional can diagnose the failure, retry transient issues, patch code-level problems, or ask the workflow owner for the missing business rule.

  - Automatic triage for failed runs
  - Retry behavior for transient failures
  - Clarification requests when a missing rule, mapping, or threshold blocks the run
  - Reminder pings so stalled runs do not get forgotten
  - Runtime picker improvements across build modes

</Update>

<Update label="April 22, 2026" tags={["New features", "Agents"]}>
  ## Multiple Modes, One Agent

  Agents gained a clearer operating model. You can move between conversational use, operator configuration, and run-focused experiences without losing thread context.

  - Mode switcher for agent surfaces
  - Chat Mode for conversational work
  - Operator Mode for configuration, workflow, tools, triggers, and policy
  - Smart thread resumption when switching modes
  - Unified prompt input across agent surfaces

  Learn more in [Dex and Agent Modes](/agents/modes).

</Update>

<Update label="April 21, 2026" tags={["New features", "Public runs"]}>
  ## Public Run Chat and Workflow Viewer Polish

  Public run pages became more useful for stakeholders who need to understand or review a run without using the full builder interface.

  - Public run chat overlay for questions about a run
  - Authenticated chat overlay for viewers who need explanations and next steps
  - Workflow viewer configuration panels tightened around run context
  - Sidebar improvements for recent agents, sessions, and toolkit shortcuts

</Update>

<Update label="April 20, 2026" tags={["Improvements", "Approvals"]}>

  Usage and tool activity moved into a clearer dashboard, while approval behavior became safer and easier to inspect.

  - Usage and tool activity grouped into dashboard tabs

  - Pending approval state made clearer
  - Bypass approvals available for stable low-risk runs

<Update label="April 10, 2026" tags={["New features", "Approvals"]}>
  ## Human Approvals Anywhere in Your Workflow

  Workflows gained explicit approval gates, making human review a first-class workflow step instead of only an instruction pattern.

  - Gate nodes can pause a workflow until a reviewer approves or rejects
  - Quorum rules let you require one approver or all listed approvers
  - Public approval links let stakeholders review context without the full builder interface

<Update label="February 6, 2026" tags={["New features", "Improvements"]}>
  ## Skills

  Upload markdown files to give your agents specialized knowledge, instructions, or playbooks. Agents reference these skills during builds and conversations.

  - Upload, view, and delete `.md` skill files per agent
  - Toggle between **Preview** (rendered markdown) and **Raw** (source text) views

  | `query_knowledge_base(query)` | Search your agent's knowledge base from workflow code |

  ## File Uploads in Chat

  You can now upload files directly to your agent during conversations for processing.

  ## Redesigned Home Page

  Clean, centered layout with a collapsible sidebar for a more focused experience.

  ## Chat-Driven Build & Test

<Update label="January 26, 2026" tags={["New features", "Improvements"]}>
  ## Slash Commands in AI Assistant

  You can now use **slash commands** in the AI Assistant chat to quickly execute agent actions without clicking buttons.

  ### Available Commands

  | Command | Description |
  |---------|-------------|
  | `/connect` | Open the integrations panel to connect tools |
  | `/build` | Build the agent workflow |
  | `/test` | Test the agent |
  | `/edit` | Return agent to draft mode for editing |
  | `/retest` | Re-run tests after making changes |
  | `/publish` | Publish the agent to live |

  ### How to Use

  1. Type `/` in the chat input to see available commands
  2. Use arrow keys to navigate, Enter to select
  3. Type to filter commands (e.g., `/con` shows only `/connect`)
  4. Press Escape to dismiss

  Commands are **context-aware**—only actions valid for your agent's current status are enabled. For example, `/publish` is only available after testing.

  ### Other Improvements

  - **Better text wrapping**: Long URLs and text now wrap properly in chat messages
  - **Subtle scrollbars**: Cleaner, more minimal scrollbar styling throughout the app

</Update>
