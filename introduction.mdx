## What is Decisional?

Decisional helps you build and operate AI agents for business workflows. You describe the work in natural language, Dex turns that intent into an agent, and the agent builds, tests, and runs its workflow across the apps and data you authorize.

<Card title="Core Philosophy" icon="lightbulb">
  **Describe what you want automated, and your agent figures out the rest.** It builds its own workflow, connects to your tools, tests itself, and runs on autopilot. When something needs to change, just tell it.
</Card>

## How It Works

<Steps>
  <Step title="Describe the Work">
    Tell Dex what you want automated in plain English. Upload any relevant documents or spreadsheets for context.
  </Step>
  <Step title="Your Agent Builds Itself">
    Decisional generates a workflow, connects to your integrations, and tests everything automatically.
  </Step>
  <Step title="Review Policy and Approvals">
    See exactly what your agent will do before it goes live. Approve actions, set approval policy, or ask Dex to change the workflow.
  </Step>
  <Step title="Let It Run">
    Your agent runs on autopilot with triggers, schedules, public run links, or on demand. Monitor progress and manage it through chat.
  </Step>
</Steps>

## What Makes Decisional Different

| | Traditional Automation | Decisional |
| --------------- | ---------------------------- | -------------------------------------- |
| **Setup** | Drag blocks, configure each step | Describe what you want in plain English |
| **Maintenance** | Rebuild when things break | Tell your agent what to change |
| **Flexibility** | Rigid, pre-defined actions | Agents reason through edge cases |
| **Documents** | Limited to API data | Reads and understands your files |
| **Completion** | Gets you partway there | Finishes the entire workflow |

<AccordionGroup>
  <Accordion title="Agents Build Their Own Workflows">
    You don't need to map out every step. Describe the outcome you want and your agent figures out how to get there, including which tools to use and what order to do things in.
  </Accordion>
  <Accordion title="Dex Manages Agents Through Chat">
    Need to change something? Tell Dex or the agent what should be different. Decisional routes that request to the right mode so simple requests stay conversational and configuration changes happen in the operator surface.
  </Accordion>
  <Accordion title="Modes Match the Job">
    Use Chat Mode for conversation, Operator Mode for configuration, and Public Mode for focused run links that can be shared without exposing the builder interface.
  </Accordion>
  <Accordion title="Approvals Are Built In">
    Agents can pause before sensitive actions, route review to a stakeholder, or bypass eligible approvals for stable low-risk work.
  </Accordion>
  <Accordion title="Reads and Reasons Over Documents">
    Upload any document and your agent understands it. PDFs, contracts, spreadsheets, emails. It uses retrieval-augmented generation to pull the right context when it needs it.
  </Accordion>
  <Accordion title="Connects to the Apps You Use">
    Team chat, CRM, email, spreadsheets, and more. Your agent works with your existing tools so nothing changes about how your team operates.
  </Accordion>
</AccordionGroup>
